## ✅ 上场前 5 分钟检查清单

1. **先唤醒后端。** 打开上面的健康检查地址，等到显示 `{"status":"UP"}`。免费实例闲置 15 分钟会休眠，冷启动要 30–90 秒——**千万别跳过**，否则第一个页面会一直转圈。
2. 并排打开**两个浏览器窗口**（或两台设备）：
   - **窗口 C** → 登录**儿童端**（`child@test.com`），这是"Leo 的平板"。
   - **窗口 P** → 登录**家长端**（`parent@test.com`），这是"Sarah 的手机"。
   - *（若提示输入 PIN，是 `1234`。）*
3. **准备好上传素材**（核心环节，别现场临时找图）：在演示机桌面放好两张图，**必须是 JPG/PNG 且小于 5MB**：
   - `Leo的杯子.jpg` —— 一个有辨识度的真实水杯实拍
   - `姜茶.jpg` —— 一杯温茶/蜂蜜水实拍
4. 确认演示家庭里**有一个视觉日程**可展示（Act 1 用）。若没有，提前在家长端日程页建一个，或用现有的日程模板。
5. 任一窗口若显示异常：打开 DevTools 控制台 → `localStorage.clear(); location.reload()`，重新登录。
6. 另开一个标签页备好**录屏备份视频**，以防断网。

## 🗣️ 开场（30 秒）—— 讲"为什么"

> "大约每 100 个孩子里就有 1 个在自闭症谱系上，他们中很多人不会说话，或者很难表达自己的需求。**VisiTalk** 用图片卡和视觉日程给他们一个'声音'，也给家长一个私密地理解孩子情绪的窗口。今天的主角是 Leo 和他的妈妈 Sarah——而且这是 Leo **生病的一天**。"

## 👦 第一幕 — Leo 的早晨（儿童端，约 2 分钟）· *窗口 C*

### 场景 ① 视觉日程：把抽象任务变成能一步步完成的步骤（B-1/B-2/B-3）
1. 在 Leo 的主页，点左下角的 **🗓️ 日程**按钮。
2. 展示当天日程——若干张图片步骤，**当前步骤高亮**，完成的步骤变绿。
3. 逐个点大 ✓ 勾选；完成最后一步 → **🎉 庆祝动画**。
   > *"对自闭症孩子来说，例行和转换是很难的。视觉日程把一个抽象的任务，变成清晰、可完成的步骤。"*

### 场景 ② "我要…"：基础 PECS 沟通（A-1/A-2）
1. 点 **← 返回**回到 Leo 的卡板。展示顶部**分类标签**：Need、Daily、Place、Eat、Drink、Play、Feel（100+ 张卡）。
2. 从分类里点出卡片，在顶部句子条组成一句话，例如 *I want apple.（我要苹果）*。
3. 点**保存/发送** → 出现在家庭聊天里。
   > *"每张卡片都同时显示**图片**和**词**——这正是 PECS 建立语言的方式。"*

## 👩 第二幕 — Leo 病了，Sarah 做"专属卡片"（家长端，约 2 分钟）· *窗口 P* ⭐核心

> **过渡旁白：** *"到了下午，Leo 开始发烧、蔫蔫的，什么都不想说，只是想喝点温的安抚一下。问题来了——"*

1. 切到家长端（如需 PIN：`1234`），进入 **PECS 卡片管理**。
2. 指着标准卡库说明痛点：
   > *"系统自带一整套标准 PECS 卡库。但生病的孩子要的是**熟悉、能让他安心的东西**——他**自己的那个杯子**、妈妈泡的那杯**姜茶**。而标准卡库里只有一个通用的卡通杯子，Leo 根本不认。"*
3. 在 **"Upload custom photo card"（上传自定义照片卡）**区域：
   - 把分类下拉选成 **Drink**。
   - 点 **Choose image**，选 `Leo的杯子.jpg`。
   - **上传完成的瞬间，卡片立刻出现在 Drink 分类里。**
4. 同样再上传 `姜茶.jpg`（分类仍选 Drink）。
   > *"支持 JPG/PNG，5MB 以内，上传即生成卡片。这一步看着简单，但它是我们产品的核心理念——**通用图标是给所有孩子的，但沟通是个性化的**。家长可以拍下孩子真正认识的物品，甚至把妈妈的脸做成'我要妈妈'。识别度完全不一样。"*

## 👦 第三幕 — Leo 用专属卡片表达（儿童端，约 1.5 分钟）· *窗口 C* ⭐闭环

1. 切回儿童端，点底部 **Drink** 分类标签。
   > *"现在 Leo 进到 Drink 分类，看到的是刚才那两张——**他自己的杯子**和**那杯姜茶**，不是陌生的卡通图。"*
2. 依次点出卡片，在句子条组成 *I want · 我的杯子 · 茶*。
3. 点**保存/发送** → 出现在家庭聊天里。
   > *"Leo 第一次清楚地表达了'我想喝茶'——而且用的是他自己认识的东西。第二幕和第三幕在这里闭环了：**家长上传 → 孩子使用**。"*

### 场景 · 表达情绪：私密日记（C-6）
1. 点右下角 **💗 日记**按钮。
2. 选一个**难受/不舒服的表情**，可在画板上**涂鸦**，点 **完成 ✓** → "已保存！"。
   > *"这是 Leo 的私密空间——而接下来这一点很关键……"*（自然过渡到 Sarah）

## 👩 第四幕 — Sarah 的记录与洞察（家长端，约 2 分钟）· *窗口 P*

### 场景 ④ 三步记录一次行为事件（C-1/C-2）
1. 注意这里**深色、专业的界面**——和孩子那边活泼的风格刻意区分。
2. 进入 **Log（记录）** → 点**强度** → 点一个**触发因素**（例如：身体不适）→ **保存**。三步搞定。
   > *"真实的情绪崩溃时刻，家长只有几秒钟——记录必须毫不费力。"*

### 场景 ⑤ 周趋势 + 预警（C-3/C-5/C-4）
1. 打开 **Report（报告）**，展示本周的**情绪强度趋势**。
   > *"这一周 Leo 的情况能看到变化——数据让进步变得可见，而不是凭感觉。"*
2. 展示 **Top 3 触发因素**：例如*巨大噪音、转换环节、身体不适*。
3. 进入 **Settings（设置）**或预警横幅：一条红色的 **3 天预警**——*某个触发连续 3 天出现*。
   > *"VisiTalk 能发现一个疲惫的家长可能错过的规律，提醒尽早干预。"*
4. 回到 Report，点 **Share / Export（分享/导出）** → 生成一个 **24 小时有效的分享链接**（给治疗师或另一位家长）。

### 场景 ⑥ 隐私边界（C-7）—— 伦理亮点
1. 在 **Settings** 里展示 **"孩子的私密日记"**：状态显示 **"✓ 今天写了"**——**永远不显示内容**。
   > *"Sarah 知道 Leo 今天表达了自己，但他画了什么、写了什么，始终是隐私。这是**设计层面的信任**——由数据库的行级安全（RLS）强制保障，而不只是界面上藏起来。"*

## 🛠️ 第五幕 — 它是真实的、工程化的产品（约 1 分钟）

1. 掏出**手机**，扫码/打开**线上地址**——"任何人现在就能用。"
2. 一张幻灯片 / 20 秒讲技术栈：
   - **前端** Vue 3 + TypeScript + Tailwind → **Vercel**
   - **后端** Spring Boot 3 + JWT → **Render**（Docker）
   - **数据库** PostgreSQL 17 → **Neon**，行级隐私隔离
   - GitHub + CI；已端到端部署并验证。

> **结束语：** *"VisiTalk 不是一个原型——它是一个已经上线的产品，能让一个生病的自闭症孩子，用他自己认识的杯子说出'我想喝茶'，完成他的一天，并且把自己的感受留给自己。"*

## 🧯 出问题时怎么办
- **首屏一直转圈** → 后端在休眠，等约 60 秒刷新（所以才要提前预热）。
- **登录被拒** → `localStorage.clear(); location.reload()`。
- **上传失败/报错** → 检查图片是否为 JPG/PNG 且小于 5MB；确认后端（`/api/uploads`）已唤醒。
- **断网** → 切到**备份视频**标签页，配合旁白讲解。
